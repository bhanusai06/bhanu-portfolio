🏆 Hackathon Winner● Live
EcoWipe
A secure web application designed to permanently delete sensitive data from recycled IT hardware. The system creates unalterable digital records and uses cryptographic verification to guarantee secure deletion. This project won First Place at the Prajwalan National Hackathon, competing against teams from over 40 colleges.
React.jsNode.jsMongoDBCryptoJSExpress
Live Demo GitHub
Architectural Rationale
Selected React and Node.js with client-side CryptoJS to enforce a zero-trust data-wiping model. By encrypting metadata locally in the browser before hardware is decommissioned and using MongoDB to store SHA-256 tamper-evident logs, we eliminate intermediate server vulnerabilities and guarantee unalterable sanitization compliance.
Key Technical Challenge
Web browser sandboxing blocks direct low-level hardware sanitization commands. To solve this without compromising the web experience, I engineered a lightweight, secure Node.js desktop daemon that bridges system-level write commands to the web frontend over localized secure WebSockets, enabling raw cryptographic passes.
Quantified Impact
Automated verification processes and certificate signing, reducing hardware validation and auditing labor from 45 minutes of manual checking down to 18 seconds of cryptographic proof generation. Selected for commercial review.

LateralShield
A security platform that uses machine learning to detect unusual network activity and identify potential security threats. It includes a simulated honeypot system that automatically redirects suspicious traffic to a safe, isolated sandbox environment to protect critical business assets.
PythonIsolation ForestFastAPIReact.jsSocket.io
GitHub Demo on request
Architectural Rationale
Designed using Python and FastAPI for high-performance packet analysis, utilizing an Isolation Forest machine learning model to detect anomalous traffic hops. Integrated Socket.io for low-latency event streaming to the React dashboard, enabling real-time topology visualization and immediate security response.
Key Technical Challenge
Under heavy traffic simulation, continuous ML model inference blocked the single-threaded FastAPI event loop, causing WebSocket connection dropouts. I resolved this by offloading model predictions to an asynchronous worker process pool using Python's ProcessPoolExecutor, keeping the connection thread clear.
Quantified Impact
Redirects 100% of detected lateral threats to an isolated, simulated honeypot sandbox within 140ms of packet anomaly detection, preventing intrusion propagation across critical assets.
